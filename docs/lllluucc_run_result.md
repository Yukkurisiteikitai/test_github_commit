# 追記
## PR
### PRとは
Pull Requestの略、コードに加えた変更を他の開発者に通知し、レビューと承認を経てメインのコードベースに統合（マージ）してもらうよう依頼する機能。  
要は **コードを他の人に見てもらって大丈夫か**を確認する機能。  
### 注意点
Pull Requestもといmergeには注意が必要な点があってそれはコンフリクト
同じファイルの別の行を編集すればいいとかをしたらどれが優先される書き込みなのかわからんってエラーが発生する**コンフリクト**という現象が発生する。
これはmain.js とかコードの関数を実行するコードの編集をするときに発生するので、意図的にこの変更の繰り返しをさせて合成をさせ続けるというよりも、main.jsでどんな関数を変更としてやるかをそのブランチで編集して調節を続けるようにすることで調節するっていうようなかんじの編集をするといい感じになりがちになる
